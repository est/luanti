## Context

需要将解析后的 Rimworld 材质数据生成为 wiki 网站。考虑两种场景：

1. **本地预览** — 开发调试用
2. **GitHub Pages 部署** — 最终发布

## Decision

### 输出格式：静态 HTML

采用 Jinja2 模板生成纯静态 HTML，无需运行时框架。

### 页面结构

```
wiki/
├── index.html              # 首页：材质总览表
├── materials/
│   ├── leather.html        # 皮革分类页面
│   ├── metal.html          # 金属分类页面
│   └── fabric.html         # 织物分类页面
├── compare/
│   └── armor.html          # 防护对比表
└── assets/
    └── style.css           # 样式
```

### 核心页面：材质总览表

按以下维度展示所有材质：

| 列 | 数据来源 | 排序 |
|----|---------|------|
| 名称 | `label` | 字母序 |
| 类型 | 父类推断 | 分组 |
| 锐器防护 | `StuffPower_Armor_Sharp` | 降序 |
| 钝器防护 | `StuffPower_Armor_Blunt` | 降序 |
| 保暖 | `StuffPower_Insulation_Cold` | 降序 |
| 散热 | `StuffPower_Insulation_Heat` | 降序 |
| 市场价值 | `MarketValue` | 升序 |
| 美观 | `Beauty` | 降序 |
| 稀有度 | `commonality` | — |

### 对比功能

- 支持按属性排序（前端 JS 或预生成多个排序版本）
- 标记最佳/最差值（高亮）

### CSS 方案

- 内联 CSS 或单文件 `style.css`
- 响应式表格，移动端友好
- 无外部 CDN 依赖（离线可用）

## Consequences

优点：

- 纯静态文件，GitHub Pages 零配置部署
- Jinja2 模板灵活，易于扩展新页面
- 无需 JavaScript 框架，加载快

缺点：

- 排序/过滤功能需要预生成多个版本或引入少量 JS
- 无搜索功能（v1 可接受）

## 后续扩展

- 添加搜索功能（引入简单的前端搜索库）
- 添加图片/图标（从游戏文件提取或使用 wiki 资源）
- 支持 Mod 数据对比
