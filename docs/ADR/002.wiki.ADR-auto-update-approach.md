## Context

需要从 MineClonia 源码自动提取物品、合成表、生物等数据，生成 Wiki 页面。

MineClonia 源码结构规整，所有物品通过 `core.register_node/craftitem/tool` 注册，
合成配方通过 `core.register_craft` 注册，数据丰富（描述、硬度、爆炸抗性、食物值等）。

## Decision

采用两阶段方案：

### Phase 1: 静态 Lua 解析（优先）

用 Python 解析 `.lua` 源文件，提取注册调用中的 table 结构。

- 能覆盖 ~95% 的手动注册数据
- 不需要编译/运行 Luanti
- CI 轻量，GitHub Actions 直接跑

### Phase 2: 运行时 dump（后续增强）

在 CI 中编译 Luanti + 加载 MineClonia，导出 `core.registered_*` 表为 JSON。

- 覆盖 100% 数据（包括动态生成的内容）
- CI 较重，但数据完整性最高

## Consequences

- Phase 1 能快速出 MVP，合成表/物品图鉴可自动同步
- Phase 2 补齐动态内容（如 `_mcl_autogroup` 自动生成的烧炼配方）
- 解析脚本放在 `scripts/` 目录
- GitHub Actions 定时检测 MineClonia release，有更新则重新生成
