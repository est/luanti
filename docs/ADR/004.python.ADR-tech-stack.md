## Context

需要选择 Python 技术栈用于：
1. 解析 Rimworld XML 数据文件
2. 提取和处理材质属性数据
3. 生成 wiki 静态页面
4. 在 GitHub Actions 中运行

约束条件：

- GitHub Actions runner 预装 Python 3.9+
- 最小依赖原则（减少供应链风险）
- 本地开发 + CI 双环境兼容

## Decision

### 核心依赖

| 用途 | 选择 | 理由 |
|------|------|------|
| XML 解析 | `xml.etree.ElementTree` (标准库) | 无依赖，满足需求 |
| 模板引擎 | `jinja2` | 业界标准，GitHub Actions 友好 |
| CLI 框架 | `argparse` (标准库) | 无依赖，简单场景够用 |
| 数据校验 | `dataclasses` (标准库) | 无依赖，类型安全 |
| 测试框架 | `pytest` | GitHub Actions 原生支持 |

### 不引入的依赖

| 不选 | 理由 |
|------|------|
| `lxml` | 需要编译 C 扩展，CI 环境可能有兼容问题 |
| `pydantic` | XML 解析场景不需要重量级校验 |
| `click` | `argparse` 足够，减少依赖 |
| `beautifulsoup4` | Rimworld 是 XML 不是 HTML |

### 项目结构

```
gamewiki/
├── src/
│   └── gamewiki/
│       ├── __init__.py
│       ├── parser.py        # XML 解析 + 继承合并
│       ├── models.py        # dataclass 数据模型
│       ├── extractor.py     # 材质/装备数据提取
│       └── generator.py     # wiki 页面生成
├── templates/               # Jinja2 模板
│   └── material.html.j2
├── tests/
│   ├── test_parser.py
│   └── test_extractor.py
├── pyproject.toml
└── .github/workflows/build.yml
```

### pyproject.toml

```toml
[project]
name = "gamewiki"
version = "0.1.0"
requires-python = ">=3.10"
dependencies = [
    "jinja2>=3.1",
]

[project.optional-dependencies]
dev = [
    "pytest>=7.0",
]

[build-system]
requires = ["setuptools>=68.0"]
build-backend = "setuptools.build_meta"
```

### GitHub Actions 集成

```yaml
# .github/workflows/build.yml
name: Build Wiki
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - run: pip install -e ".[dev]"
      - run: pytest
      - run: python -m gamewiki --data-dir ./rimworld-data --output-dir ./wiki
      - uses: actions/upload-artifact@v4
        with:
          name: wiki
          path: ./wiki/
```

## Consequences

优点：

- 只有 1 个第三方依赖（jinja2），供应链风险极低
- 标准库 XML 解析足够处理 Rimworld 的 XML 结构
- GitHub Actions 零配置，`pip install` 即可运行
- 本地开发和 CI 环境完全一致

缺点：

- 标准库 XML 解析器不支持 XPath 2.0（但 Rimworld 的查询需求简单）
- 没有 XML schema 校验（可通过测试覆盖）
