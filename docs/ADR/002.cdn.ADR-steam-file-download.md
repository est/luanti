## Context

项目目标是自动从 Steam 下载游戏包，解压英雄资料文件，生成 wiki 网站。第一步需要确认：能否从 Steam CDN 下载指定游戏的指定文件（而非完整安装游戏）。

核心需求：

- 给定 Steam App ID，下载游戏内特定文件（如英雄数据）
- 尽量不需要完整安装游戏
- 适合 CI/CD 自动化场景

## 调研结论

**可行，但有条件。**

### 1. Steam 内容分发架构

Steam 将游戏内容组织为 **Depot**（仓库），每个 Depot 有一个 Depot ID，包含一组文件。一个 App 可以有多个 Depot（按平台、语言、内容类型拆分）。CDN 按 chunk 分发，需要 manifest 来描述文件与 chunk 的映射关系。

关键概念：

| 概念 | 说明 |
|------|------|
| App ID | 游戏唯一标识（如 Dota 2 = 570） |
| Depot ID | 内容仓库标识（如 571 = Dota 2 Content） |
| Manifest | 描述 Depot 内文件树和 chunk 映射的清单文件 |
| Chunk | CDN 最小传输单元 |

### 2. 可用工具对比

#### 方案 A: DepotDownloader（推荐）

- **仓库**: https://github.com/SteamRE/DepotDownloader
- **语言**: C# (.NET 8.0)
- **Stars**: 3.2k，活跃维护（最新 3.4.0，2025-05）
- **关键能力**:
  - 支持下载指定 App 的指定 Depot
  - 支持 `-filelist` 参数，**只下载文件列表中匹配的文件**（核心需求）
  - 支持 `-manifest-only` 只下载 manifest 不下载内容
  - 支持匿名账户下载部分免费游戏
  - 支持正则匹配文件路径（`regex:` 前缀）
  - 可通过 Homebrew 安装：`brew tap steamre/tools && brew install depotdownloader`

示例命令：

```bash
# 只下载指定文件
./DepotDownloader -app 570 -depot 571 -filelist hero_files.txt

# hero_files.txt 内容示例：
# scripts/npc/heroes/hero_antimage.txt
# scripts/npc/heroes/hero_axe.txt
# regex:scripts/npc/heroes/hero_.*\.txt
```

**优势**: 成熟稳定，支持文件级过滤，CLI 友好，适合 CI/CD。
**劣势**: 需要 .NET 运行时；付费游戏需要 Steam 账号密码。

#### 方案 B: SteamCMD

- **来源**: Valve 官方命令行工具
- **关键能力**:
  - `download_depot <appid> <depotid>` 下载整个 Depot
  - 支持指定 manifest 版本
  - 官方维护
- **限制**:
  - **不支持文件级过滤**，只能下载整个 Depot
  - 付费游戏需要账号+已购买
  - 需要交互式登录或用 `+login anonymous`

**优势**: 官方工具，稳定可靠。
**劣势**: 无法只下载部分文件，下载量大。

#### 方案 C: ValvePython/steam (Python)

- **仓库**: https://github.com/ValvePython/steam
- **语言**: Python
- **Stars**: 1.3k
- **关键能力**:
  - `CDNClient` 类可直接访问 Steam CDN depot
  - `WebAPI` 接口查询 App/Depot 信息
  - `steamctl` CLI 工具封装（https://github.com/ValvePython/steamctl）
- **限制**:
  - 需要自己实现文件过滤逻辑
  - 文档较少，需要理解 Steam 协议
  - 依赖 gevent

**优势**: Python 生态，可深度定制。
**劣势**: 需要更多开发工作，不如 DepotDownloader 开箱即用。

#### 方案 D: SteamKit2 (底层库)

- **仓库**: https://github.com/SteamRE/SteamKit
- **语言**: C# (.NET)
- **Stars**: 3.1k
- **说明**: DepotDownloader 的底层库。除非需要完全自定义协议交互，否则直接用 DepotDownloader。

### 3. 认证与权限

| 场景 | 认证要求 |
|------|---------|
| 免费游戏（如 Dota 2） | 匿名账户即可 |
| 付费游戏 | 需要 Steam 账号 + 已购买该游戏 |
| 受限内容（DLC 等） | 需要账号 + 对应 DLC 所有权 |
| 旧版本 manifest | 可能被开发者封锁，需要登录 |

对于 wiki 生成场景，通常只需要免费游戏或已有账号的游戏数据。

### 4. 法律与 ToS

- 下载自己已购买的游戏文件用于个人/非商业用途，通常不违反 Steam ToS
- **不应**将下载的游戏文件重新分发
- Valve 对 reverse engineering 的态度是灰色地带，但 DepotDownloader 等工具已存在多年未被取缔
- 用于生成 wiki 属于信息提取，风险较低

### 5. CI/CD 集成方案

```bash
# 典型 CI/CD pipeline
# 1. 安装 DepotDownloader
brew install depotdownloader  # macOS
# 或下载 release binary

# 2. 准备文件列表
echo "regex:scripts/npc/heroes/.*" > filelist.txt

# 3. 下载（Dota 2 示例）
DepotDownloader -app 570 -depot 571 -filelist filelist.txt -dir ./game_data

# 4. 解析文件，生成 wiki
python generate_wiki.py ./game_data
```

注意事项：

- CI 环境中 Steam 登录可用环境变量传入（`-username $STEAM_USER -password $STEAM_PASS`）
- 2FA 需要用 `-remember-password` 预先保存 session，或使用 Steam Guard 专用密码
- 可配合 Lancache 做本地 CDN 缓存加速

## Decision

采用 **DepotDownloader** 作为首选方案：

1. 支持文件级过滤（`-filelist`），满足只下载英雄数据的需求
2. 成熟稳定，社区活跃
3. CLI 友好，易于 CI/CD 集成
4. 跨平台（.NET 8.0）

备选方案：ValvePython/steam，当需要深度定制或纯 Python 栈时考虑。

## Consequences

优点：

- 可以精确下载需要的文件，不浪费带宽和存储
- 自动化友好，可集成到 CI/CD pipeline
- 免费游戏无需购买

缺点：

- 需要 .NET 8.0 运行时环境
- 付费游戏需要维护 Steam 账号凭据（安全风险）
- Valve 可能随时更改协议或封锁工具（长期风险）
- 需要先确定目标游戏的 Depot ID 和文件路径结构

## 下一步

1. 选定目标游戏，用 SteamDB 查询其 Depot ID 和文件结构
2. 安装 DepotDownloader 做 PoC 验证
3. 确认目标文件的具体路径和格式
4. 设计文件解析和 wiki 生成 pipeline
