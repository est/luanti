## Context

需要解析 Rimworld 的游戏数据文件，提取材质属性（皮革、毛皮、金属等）用于生成 wiki。

Rimworld 的数据存储在本地 Steam 安装目录：

```
~/Library/Application Support/Steam/steamapps/common/RimWorld/RimWorldMac.app/Data/
├── Core/Defs/           # 基础游戏数据
├── Biotech/Defs/        # Biotech DLC
├── Royalty/Defs/        # Royalty DLC
└── Ideology/Defs/       # Ideology DLC
```

Steam App ID: **294100**

## 数据结构分析

### XML Defs 体系

Rimworld 使用 XML 定义所有游戏数据，核心机制：

1. **继承** — `ParentName` 属性实现继承，子元素覆盖父元素的值
2. **抽象基类** — `Abstract="True"` 定义模板（如 `LeatherBase`）
3. **Patch 操作** — DLC 通过 `PatchOperationReplace` / `PatchOperationAdd` 等修改 Core 定义

### 材质关键属性

从 `Items_Resource_Stuff_Leather.xml` 分析，每种皮革包含：

| 属性 | XML 路径 | 说明 |
|------|---------|------|
| defName | `defName` | 唯一标识（如 `Leather_Thrumbo`） |
| label | `label` | 显示名（如 `thrumbofur`） |
| description | `description` | 描述文本 |
| Armor Sharp | `statBases/StuffPower_Armor_Sharp` | 锐器防护 |
| Armor Blunt | `statBases/StuffPower_Armor_Blunt` | 钝器防护 |
| Armor Heat | `statBases/StuffPower_Armor_Heat` | 热伤害防护 |
| Insulation Cold | `statBases/StuffPower_Insulation_Cold` | 保暖 |
| Insulation Heat | `statBases/StuffPower_Insulation_Heat` | 散热 |
| MarketValue | `statBases/MarketValue` | 市场价值 |
| MaxHitPoints | `statBases/MaxHitPoints` | 耐久 |
| Beauty | `stuffProps/statFactors/Beauty` | 美观系数 |
| Flammability | `statBases/Flammability` | 易燃性 |
| color | `stuffProps/color` | 颜色 (R,G,B) |
| commonality | `stuffProps/commonality` | 稀有度 |

### 继承示例

```xml
<!-- 抽象基类，定义默认值 -->
<ThingDef Abstract="True" Name="LeatherBase" ParentName="ResourceBase">
  <statBases>
    <StuffPower_Armor_Sharp>0.81</StuffPower_Armor_Sharp>
    <StuffPower_Insulation_Cold>16</StuffPower_Insulation_Cold>
  </statBases>
</ThingDef>

<!-- 子类，只覆盖差异部分 -->
<ThingDef ParentName="LeatherBase">
  <defName>Leather_Thrumbo</defName>
  <label>thrumbofur</label>
  <statBases>
    <StuffPower_Armor_Sharp>2.08</StuffPower_Armor_Sharp>  <!-- 覆盖 -->
    <StuffPower_Insulation_Cold>34</StuffPower_Insulation_Cold>  <!-- 覆盖 -->
  </statBases>
</ThingDef>
```

## Decision

采用**两阶段解析**方案：

### 阶段一：构建完整 Def 树

1. 解析所有 `Data/` 下的 XML 文件
2. 处理继承链：遇到 `ParentName` 时，合并父节点的属性
3. 处理 `Abstract="True"` 的模板节点（不输出，仅供继承）
4. 按 `defName` 索引所有非抽象 Def

### 阶段二：应用 Patch

1. 解析 DLC 和 Mod 的 Patch XML
2. 按 `Operation Class` 执行对应的修改操作：
   - `PatchOperationReplace` — 替换节点
   - `PatchOperationAdd` — 添加节点
   - `PatchOperationRemove` — 删除节点
   - `PatchOperationSequence` — 顺序执行多个操作

### 解析策略

- 使用 Python 标准库 `xml.etree.ElementTree`（无需额外依赖）
- 递归合并继承链，处理多层继承
- 输出标准化的 JSON/字典结构

## Consequences

优点：

- 纯 Python 标准库，无外部依赖，GitHub Actions 原生支持
- 本地文件直接解析，不需要通过 Steam CDN 下载
- 两阶段设计正确处理继承和 DLC 覆盖

缺点：

- 需要实现继承链解析逻辑（非平凡）
- Patch 操作类型较多，需要完整实现
- Mod 的 Patch 可能引入额外复杂度（v1 暂不支持 Mod）

## 实现要点

```python
# 解析流程伪代码
def parse_all_defs(data_dir: Path) -> dict[str, Def]:
    defs = {}
    # 1. 扫描所有 XML，按 defName 索引
    for xml_file in data_dir.rglob("*.xml"):
        tree = ET.parse(xml_file)
        for elem in tree.findall(".//ThingDef"):
            defs[elem.find("defName").text] = elem

    # 2. 解析继承链
    for defname, elem in defs.items():
        if parent := elem.get("ParentName"):
            merge_with_parent(elem, defs[parent])

    # 3. 过滤掉 Abstract 节点
    return {k: v for k, v in defs.items()
            if v.get("Abstract") != "True"}
```
